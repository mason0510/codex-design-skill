# 使用示例

## 示例 1：高保真原型

用户：
> 帮我做一个 B2B SaaS 仪表盘原型，要求高保真、HTML 交付、给我 3 个视觉方向。

期望行为：
- 先问受众、品牌、现有设计系统、信息密度要求
- 明确 3 个方向分别想比较什么
- 最终至少交一个可运行主文件，再给切换方向的方式

## 示例 2：HTML 幻灯片

用户：
> 帮我把这份 PRD 做成 8 页 HTML 幻灯片，偏产品汇报风格。

期望行为：
- 先确认受众、时长、语气
- 用演示文稿思维组织节奏，而不是网页思维
- 交付前检查加载和翻页逻辑

## 示例 3：基于现有 UI 重建

用户：
> 按这个代码库现有设计系统，重建设置页并给我两个布局变体。

期望行为：
- 先读现有颜色、字体、间距、组件
- 说出观察到的视觉语汇
- 输出对齐现有系统的设计，而不是另起炉灶

## 示例 4：动效 demo

用户：
> 我想做一个产品能力展示页，不是官网，要更像动效演示短片。

期望行为：
- 先确认它是“动效展示”而不是“营销页”
- 重点处理节奏、分镜、状态保持
- 避免所有内容一次性同时运动

## 示例 5：只修小样式时不要误用

用户：
> 帮我把这个表单按钮右边距调成 12px。

期望行为：
- 不要启用整套设计交付 skill
- 直接按普通前端修复处理
