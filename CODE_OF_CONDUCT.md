# Contributor Code of Conduct

## Our Commitment

为了让 `codex-design-skill` 社区保持开放、务实、可协作，
我们承诺把参与体验做得尽量清晰、尊重、直接。

无论经验、身份、背景如何，所有贡献者都应能在这里围绕事实、设计判断和实现边界进行讨论。

## Our Standards

### 鼓励的行为

- 基于事实、样例和可复用经验讨论问题
- 对设计判断、仓库边界和实现方式进行直接但专业的反馈
- 尊重不同背景和经验层次的贡献者
- 在提出批评时，给出可执行的替代方案
- 发现敏感信息、专有残留或不适合公开的内容时，主动提醒维护者

### 不接受的行为

- 人身攻击、羞辱、威胁或恶意骚扰
- 用歧视性、侮辱性语言攻击任何人
- 故意引战、钓鱼、刷屏或恶意消耗维护者时间
- 在公开讨论中泄露私人信息、密钥、内部路径、未公开资产
- 把仓库讨论引向与项目无关的攻击、营销或政治斗争

## Enforcement Responsibilities

维护者有权删除、编辑或拒绝不适合公开社区的评论、Issue、PR、文档和代码。

## Scope

本行为准则适用于：

- 仓库 Issues / PRs / Discussions
- 文档评论与 review 讨论
- 项目相关的其他公开协作场景

## Enforcement

如果你遇到违反行为准则的内容，请联系维护者并附带上下文。

维护者会根据情况采取以下措施：

1. 提醒或警告
2. 锁定讨论
3. 关闭 Issue / PR
4. 暂停或永久禁止参与仓库协作
