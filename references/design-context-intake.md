# 设计上下文收集指南

## 什么时候读

当任务是新的、模糊的，或用户没有给足上下文时，优先读这份文件。

## 最少要问清楚的 5 件事

1. 交付物是什么：原型、幻灯片、动效 demo 还是展示页？
2. 谁会看：老板、客户、内部产品、工程团队还是终端用户？
3. 保真度到什么程度：线框、中保真还是高保真？
4. 是否已有设计系统、组件库、截图、Figma、现有代码？
5. 用户希望比较哪些维度：视觉、交互、文案、信息密度还是动效？

## 上下文优先级

```text
现有代码 / 组件
→ 设计系统 / 品牌规范
→ 截图 / Figma / 参考页
→ 用户口头描述
→ 自主提出原创方向
```

## 实战规则

- 没有上下文时，不要直接构造一整套品牌视觉
- 如果用户已有产品，应尽量对齐现有视觉语汇
- 在开始设计前，最好先说出你观察到的规律

## 观察维度

- 颜色：主色、辅助色、明暗关系
- 字体：标题感、正文感、语气轻重
- 版式：稀疏还是紧凑
- 卡片：圆角、边框、阴影方式
- 动效：克制还是明显
