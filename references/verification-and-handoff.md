# 验证与交付收尾

## 交付前最低检查

### 1. 加载检查
- 页面能打开
- 没有明显白屏
- 没有缺失关键脚本导致的直接报错

### 2. 控制台检查
- 无明显报错
- 无阻断主流程的异常

### 3. 交互检查
- 主要按钮可点击
- 主要翻页 / 切换逻辑可用
- tweak / 变体切换后不崩

### 4. 视觉检查
- 主次关系清楚
- 对齐没有明显事故
- 留白不是靠硬塞内容解决

## 交付时应该说明什么

最少说清：
- 当前交付物是什么
- 里面有哪些可比较方向
- 下一步最值得用户拍板的点是什么

## 不要这样收尾

- 页面没跑就说“已完成”
- 只描述想法，不交付文件
- 只给一个方向却说“已经探索完毕”
