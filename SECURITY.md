# Security Policy

## Supported scope

`codex-design-skill` 是一个以文档、模板和轻量校验脚本为主的开源仓库。

这里最值得优先报告的安全 / 边界问题包括：

- 仓库中误放了敏感信息
- `validate.sh` 对敏感内容校验存在明显绕过路径
- 文档、模板或示例中包含不应公开的专有残留
- 社区模板或自动化流程会诱导用户提交敏感数据

## Reporting a vulnerability

请不要先公开提 Issue。

推荐流程：

1. 准备最小复现或证据
2. 明确说明受影响文件
3. 说明你判断它为什么不适合公开
4. 通过私下渠道联系维护者

如果你暂时无法联系到维护者，至少不要把密钥、私人路径、账号、cookie 或闭源内容直接公开贴在仓库里。

## Response expectations

维护者目标：

- 在合理时间内确认收到问题
- 先判断是否确实属于公开仓库边界问题
- 若问题成立，优先做删除、替换、修复或补规则
- 必要时在 `CHANGELOG.md` 记录修复结果

## Disclosure policy

在问题未被确认和处理前，请不要公开扩散细节。

尤其是涉及以下内容时：
- 私有 prompt
- 本机路径
- 私人邮箱
- 真实 secret / token / key
- 内部品牌资产与未公开文档
